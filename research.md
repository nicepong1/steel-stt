# 💡 STT (Speech-to-Text) 정확도 및 성능 향상을 위한 심층 연구 가이드 (research.md)

이 문서는 최신 음성 인식 논문 및 시스템 구축 사례를 바탕으로 STT의 인식률(Accuracy)과 강건성(Robustness)을 극대화하기 위한 구체적인 방법론을 정리한 문서입니다. 바이브 코딩 시 아키텍처 설계, 데이터 전처리, 학습 전략, 디코딩 최적화의 지침으로 사용합니다.

## 1. 한국어 특화 아키텍처 최적화 (Korean-Specific Optimization)

*   **자모(Jamo) 단위 분리 모델링:** 한국어 음절(11,172개)을 그대로 출력 노드로 사용하면 수렴이 어렵고 발음의 모호성(예: '간'과 '궁'의 초성 동일성)을 학습하기 어렵습니다. 텍스트를 초성, 중성, 종성으로 분리하여 학습하면 출력 집합을 약 40~58개로 대폭 줄일 수 있어 정확도와 학습 속도가 크게 향상됩니다 [1-3].
*   **유한 오토마타(Finite Automata, FA) 기반 언어 모형 적용:** 자모 분리 방식 사용 시, 디코딩 과정에서 초성-중성-종성의 순서가 어긋나는 문제를 막기 위해 상태 전이 규칙을 정의한 FA 언어 모형을 적용하면 올바른 한국어 문자열만 탐색 공간에 남게 되어 WER(단어 오류율)이 감소합니다 [4-6].
*   **L Transducer (발음 사전 WFST) 활용:** "학교"가 "학꾜"로 발음되는 것처럼 문맥에 따른 발음 변형 규칙을 End-to-End 모델이 스스로 학습하기는 어렵습니다. G2P(Grapheme-to-Phoneme) 모델을 이용해 가능한 발음열을 WFST(가중 유한 상태 트랜스듀서) 형태의 Lexicon Transducer로 구축하고, 모델 출력과 결합(Composition)하여 탐색 공간을 제한하면 오인식을 획기적으로 줄일 수 있습니다 [7-11].

## 2. 최신 음향 모델 아키텍처 설계 (Acoustic Model Architecture)

*   **Conformer 도입:** 시퀀스 모델링에 강한 Transformer의 Self-attention과 지역적(Local) 음향 특징을 잘 추출하는 Convolution을 결합한 **Conformer** 아키텍처가 현재 가장 우수한 성능을 보입니다 [3, 12-14].
*   **Joint CTC-Attention 구조:** 음성 프레임과 텍스트 간의 엄격한 정렬을 학습하는 CTC(Connectionist Temporal Classification) 손실 함수와, 문맥의 의미를 파악하는 Attention 디코더를 결합하여 다중 작업 학습(Multi-task Learning)을 수행하면 두 모형의 단점을 상호 보완하여 정확도가 향상됩니다 [15-18].
*   **하이브리드 어텐션 (위치 인식 + 콘텐츠):** 단순히 음향 특징(Content)만으로 어텐션을 계산하면 긴 발화에서 유사한 발음이 반복될 때 길을 잃습니다. 이전 스텝의 어텐션 가중치(Alignment)에 컨볼루션 필터를 적용하여 위치 정보(Location-aware)를 추출해 더하면 긴 문장에서도 강건하게 인식합니다 [19-21].

## 3. 데이터 전처리 및 강건성(Robustness) 확보 전략

*   **SpecAugment 기법:** 수집된 음성 데이터가 부족하더라도 스펙트로그램 이미지에서 시간축(Time Masking)과 주파수축(Frequency Masking)을 무작위로 가리는 방식을 사용하면, 과적합을 막고 부분적인 소음이나 데이터 유실에 매우 강건한 모델이 만들어집니다 [22-24].
*   **다중 조건 학습 및 노이즈 증강:** 실제 환경의 배경 소음과 반향(Reverberation)이 섞인 데이터를 인위적으로 합성하여 학습시킴으로써(Multi-condition training) 현장 소음에 대한 내성을 확보해야 합니다 [25, 26].
*   **커리큘럼 학습 (Curriculum Learning):** 처음에는 노이즈가 없는 깨끗한 음성(High SNR)으로 학습을 시작하고, 점진적으로 소음이 심한 어려운 데이터를 투입하면 수렴 속도와 최종 인식 정확도가 모두 개선됩니다 [27].

## 4. 손실 함수 및 학습(Training) 최적화

*   **라벨 스무딩 (Label Smoothing):** 정답 라벨에 1.0의 확률을 주는 대신, 작은 확률(예: 0.05~0.2)을 다른 클래스에 분배하여 모델이 과도하게 확신(Overconfidence)하는 것을 방지하면 빔 탐색 시 성능이 크게 개선됩니다 [24, 28].
*   **베이즈 딥러닝 (Variational Bayes - VB Dropout):** 점 추정치 대신 사후 분포를 추정하는 베이즈 신경망을 적용하고, 몬테카를로 추정치를 도출하면 불확실성을 관리할 수 있어 과적합을 방지하고 정확도(특히 Attention 모형에서)를 높일 수 있습니다 [16, 29-31].
*   **MWER (Minimum Word Error Rate) 파인튜닝:** 일반적인 Cross-Entropy 학습이 끝난 후, 모델이 뱉어낸 여러 가설 중 실제 단어 오류율(WER)을 최소화하는 방향으로 디코더를 미세 조정(Sequence-discriminative fine-tuning)하면 긴 오류의 연속 발생(Error streaks)을 줄일 수 있습니다 [18, 32, 33].

## 5. 디코딩 및 실시간(Streaming) 대응 기술

*   **컨텍스트 바이어싱 (Contextual Biasing / CLAS):** 철강업계의 특정 고객사 이름, 전문 강종 이름 등은 일반 데이터로 학습하기 어렵습니다. 디코딩 시 다이나믹하게 특정 단어 목록을 인코더와 디코더의 어텐션 모듈에 주입(Biasing)하여 해당 단어들의 인식 확률을 극대화해야 합니다 [34, 35].
*   **언어 모델(LM) 융합 (Shallow Fusion):** 디코딩 시 빔 탐색(Beam Search) 과정에서 음향 모델의 점수와 텍스트 대용량 코퍼스로 별도 학습된 언어 모델(N-gram 또는 Transformer LM)의 점수를 선형 결합하면 문법적 오류를 획기적으로 줄일 수 있습니다 [36-38].
*   **체크포인트 평균화 (Parameter Averaging):** 학습이 완료된 후 마지막 N개의 에포크(Epoch) 가중치(Checkpoint)를 평균 내어 최종 모델을 생성하면, 더 넓은 최적점을 찾아 일반화(Generalization) 성능이 향상됩니다 [32].
*   **빠른 음성 활동 감지 (VAD) 및 청크 처리:** 실시간 스트리밍 환경에서는 단순하고 빠른 VAD를 적용하여 침묵 구간을 끊어주고(Endpointing), 오디오를 고정된 청크(Chunk) 단위로 쪼개어 부분 어텐션(Chunked Attention)을 수행해야 지연(Latency) 없이 매끄러운 텍스트를 출력할 수 있습니다 [39-42].

## 6. 차세대 패러다임: 표현 학습과 다중 작업

*   **자기 지도 학습 (Self-Supervised Learning, 예: wav2vec 2.0):** 라벨이 없는 대량의 음성 데이터를 Contrastive task 등으로 사전 학습(Pre-training)하여 음성의 핵심 표현(Representation)을 모델이 스스로 깨치게 한 뒤, 적은 양의 철강 도메인 데이터로 파인튜닝하면 극강의 데이터 효율성과 높은 인식률을 확보할 수 있습니다 [43-45].
*   **형태소/음소/자소 다중 예측 (Refinement):** 인코더의 결과를 바탕으로 음소와 자소(또는 형태소)를 독립적으로 예측하게 한 뒤, 각 결과를 서로 교환하여 다시 디코딩하는 Refinement 구조를 취하면 언어적 일관성이 높아져 오류율이 감소합니다 [46-48].
