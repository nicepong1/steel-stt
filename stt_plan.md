# 프로젝트 기획서: 철강회사 맞춤형 실시간 STT(음성 인식) 웹앱 프로토타입

## 1. 프로젝트 개요 (Project Overview)
- **목표:** 고객의 음성 주문(예: "H빔 10톤 주문할게")을 스마트폰이나 PC 마이크로 입력받아, 실시간으로 텍스트로 변환하여 화면에 보여주는 웹앱 구축. (향후 자동 거래명세서 작성을 위한 1단계 타당성 검증)
- **주요 철학:** 기획과 코딩의 완벽한 분리. 사용자가 이 문서를 통해 검토하고 승인하기 전까지는 절대 코드를 먼저 짜지 않는다.

## 2. 주요 기능 (Core Features)
1. **마이크 접근 및 음성 스트리밍:** 모바일 및 PC 브라우저 마이크 권한 호출 및 실시간 음성 데이터 캡처 기능.
2. **직관적 텍스트 시각화 (Visual Feedback):**
   - 사용자가 말하는 도중에는 회색 텍스트(가설 텍스트, Partial result)로 화면에 실시간 표시.
   - 발화가 끝나면(침묵 감지 시) 검은색 텍스트(최종 확정, Final result)로 화면에 고정.
3. **[옵션] 철강 키워드 하이라이팅 기능:**
    - 화면에 출력된 텍스트 중 철강 관련 주요 품목(예: 철근, 강판, H빔)과 숫자(수량/규격)를 정규식 등으로 감지하여 색상을 다르게 하이라이팅.
4. **[옵션] 결과 검수 및 수동 수정 UI:**
    - STT로 텍스트화된 주문 내용에 특정 철강 전문 용어 오인식이 있을 경우, 사용자가 터치하여 수동으로 수정할 수 있는 입력폼 지원.

## 3. 기술 스택 (Tech Stack)
- **Frontend (Web/PWA):** Next.js, React, TailwindCSS, Web Audio API (마이크 스트리밍용)
- **Backend (Streaming Server):** Python, FastAPI, WebSockets (지연 없는 양방향 오디오/텍스트 통신)
- **STT API:** Google Cloud Speech-to-Text API (Streaming 모드 활용)
   *(※ 향후 자체 Conformer/RNN-T 음향 모델로 교체 예정이므로 연동부는 모듈화하여 설계)*
- **Deploy:** GitHub & Vercel (Frontend 웹 배포)

## 4. 단계별 구현 전략 (Step-by-Step Implementation)
*주의: 각 Phase가 끝날 때마다 사용자의 확인을 받고, 허락이 떨어져야만 다음 Phase로 넘어간다.*

- **Phase 1 (환경 설정 및 통제 규칙 부여):**
   - GitHub 저장소 초기화.
  - `/init` 명령어를 통해 AI를 위한 프로젝트 설명서인 `CLAUDE.md` 생성 (WebSocket 기반 구조, 보안 키 노출 금지 등 핵심 규칙 기재).
- **Phase 2 (백엔드 WebSocket & STT API 연동):**
   - FastAPI 프로젝트 세팅 및 WebSocket 엔드포인트 구현.
  - 외부 STT API와 연동하여, 들어오는 오디오 청크를 텍스트로 변환하여 즉시 클라이언트로 반환하는 로직 구현 및 사전 검증.
- **Phase 3 (프론트엔드 오디오 캡처 및 직관적 시각화):**
   - Next.js 초기화, 마이크 권한 요청 및 스트림 캡처 컴포넌트 구현.
  - **[핵심]** 프론트엔드에서 캡처한 오디오를 백엔드 웹소켓으로 보내고, 수신된 텍스트(Partial/Final 상태)를 화면에 회색/검은색으로 매끄럽게 렌더링하는 로직 완성.
- **Phase 4 (고급 필터링 및 수동 수정 구현):**
   - 철강 주요 단어 및 숫자 하이라이팅 로직 추가.
  - 화면 터치를 통한 오인식 텍스트 수정 기능 완성.
- **Phase 5 (최종 통합 및 핸드오프):**
   - 전체 STT 주문 접수 플로우 테스트 및 Vercel 배포.
  - 현재까지의 구현 상태와 향후 자체 음향 모델 도입을 위한 인수인계 사항을 `HANDOFF.md`로 정리.
